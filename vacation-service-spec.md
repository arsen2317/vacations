# Техническое описание задачи: редизайн сервиса отпусков

> Версия: 0.1 (драфт)  
> Статус: в работе

---

## 1. Цель

Создать удобный сервис на внутреннем портале, который позволяет:
- планировать ежегодный оплачиваемый отпуск в рамках кампании
- управлять остатком дней отпуска
- подавать заявки на внеплановый отпуск
- согласовывать и отслеживать заявки

Ключевая UX-задача: чётко разграничить сценарии планового и внепланового отпуска, чтобы пользователи понимали, каким путём им нужно идти в каждой конкретной ситуации.

---

## 2. Ключевые сущности

### 2.1 Кампания по планированию отпусков

Кампания — ограниченный по времени период, в течение которого сотрудники распределяют дни ежегодного оплачиваемого отпуска на следующий год.

- Привязана к конкретному году (например, «Кампания 2026»)
- Имеет два состояния: **активна** / **не активна**
- Запускается HR-админом
- Вне активной кампании функция планирования недоступна

**UX-требование:** Пользователь всегда должен чётко понимать, идёт ли кампания сейчас, и какие действия ему доступны в текущем состоянии. Статус кампании должен быть виден на главном экране сервиса и влиять на пустые стейты и подсказки.

> ⚠️ К уточнению: задаёт ли HR-админ дату окончания кампании, или длительность фиксированная?

### 2.2 Типы дней отпуска

| Тип | Описание |
|-----|----------|
| Основной ежегодный оплачиваемый | Стандартные дни отпуска |
| Дополнительный | 3 дня, есть не у всех сотрудников |

Баланс отображается раздельно: «5 дней основного отпуска» и «3 дня дополнительного отпуска».

> ⚠️ К уточнению: дополнительные 3 дня — обязательно ли их распределять в рамках кампании, или их можно брать только через внеплановый?  
> ⚠️ К уточнению: что происходит с неиспользованными днями в конце года — сгорают, переносятся, выплачиваются?

### 2.3 Типы внеплановых заявок

| Тип | Списываются дни? |
|-----|-----------------|
| Ежегодный оплачиваемый | Да — из баланса основного отпуска (или авансом) |
| Без сохранения зарплаты | Нет |
| Учебный оплачиваемый | Нет |
| Учебный без сохранения зарплаты | Нет |

> ⚠️ К уточнению: есть ли лимит на уход в минус при взятии авансом? Нужно ли показывать предупреждение в UI?

### 2.4 Статусная модель заявки

```
Черновик → На согласовании → Согласована
                           → Отклонена
Согласована → Отменена
```

- **Черновик** — создан, не отправлен на согласование
- **На согласовании** — отправлен, ожидает ответа согласующих
- **Согласована** — все согласующие одобрили
- **Отклонена** — хотя бы один согласующий отклонил (требуется комментарий)
- **Отменена** — сотрудник отменил заявку до её наступления

### 2.5 Согласующие

- **Основной согласующий** — руководитель сотрудника, подставляется автоматически
- **Дополнительный согласующий** — может быть назначен сотрудником вручную при подаче заявки

> ⚠️ К уточнению: параллельное или последовательное согласование при нескольких согласующих?  
> ⚠️ К уточнению: если один согласующий отклонил — заявка сразу отклонена или ждёт второго?  
> ⚠️ К уточнению: что если у сотрудника нет руководителя в системе — кто становится согласующим?

### 2.6 Подсчёт дней

- Выходные (суббота, воскресенье) **включаются** в дни отпуска
- Государственные праздники **не включаются** в дни отпуска

## 3. Роли пользователей

### 3.1 Сотрудник

#### Планирование отпуска (в рамках кампании)

- Выбирает дни отпуска на следующий год в интерфейсе кампании
- Сохраняет черновик — можно начать и вернуться позже
- Отправляет черновик на согласование

**Ограничения при планировании:**
- Все дни отпуска должны быть распределены перед отправкой
- Допустимо дробление на части
- Один из отрезков обязан быть не менее 14 дней

#### Работа с согласованным планом

- Подтверждает ознакомление с согласованным планом отпуска
- Может перенести запланированный отрезок:
  - Не позднее чем за 10 дней до его начала
  - Доступно **2 переноса на один отрезок**
  - Перенос требует согласования

#### Внеплановые заявки

- Подаёт заявки на внеплановый отпуск (4 типа — см. п. 2.3)
- При подаче может оставить комментарий
- Может назначить дополнительного согласующего

**Новый сотрудник:** если сотрудник пришёл после старта кампании, он не участвует в ней. Свои дни он может использовать только через подачу внепланового отпуска типа «ежегодный оплачиваемый».

#### Просмотр и управление

- Видит список всех своих заявок, включая прошедшие, с фильтрацией по статусу
- Просматривает подробную карточку заявки (модальное окно)
- Видит баланс доступных дней (основной + дополнительный, раздельно)
- Может отменить заявку до её наступления (автоматически, без согласования)
- Может редактировать черновики

> ⚠️ К уточнению: нужно ли согласование при отмене согласованного внепланового отпуска?

#### Календарь коллег

- Просматривает отпуска коллег в календарном виде
- По умолчанию отображается своя команда
- Можно добавить коллег из других команд через поиск
- Можно включить отображение черновиков (опционально)
- Система предупреждает о пересечении отпуска с коллегами из команды

#### Личный виджет отпуска *(одобрено)*

- Countdown до начала ближайшего согласованного отпуска — отображается на видном месте
- Виджет с персональной обложкой: пользователь может загрузить своё изображение (место назначения, фото и т.д.)
- По умолчанию — заглушка-графика

---

### 3.2 Руководитель

Всё то же, что у сотрудника, плюс:

#### Входящие заявки на согласование

- Видит заявки от своих подчинённых
- Фильтрация по подразделениям (у руководителя может быть несколько подразделений)
- Поиск по ФИО сотрудника
- Может согласовать или отклонить заявку
- При отклонении — комментарий обязателен

#### Отчётность

- Выгрузка отчёта по графикам отпусков сотрудников в формате xlsx

---

### 3.3 HR-админ

Всё то же, что у сотрудника, плюс:

#### Управление кампанией

- Запускает кампанию по планированию отпусков на конкретный год

#### Декабрьская опция

- Может включить опцию «разрешить отмену в декабре»
- При активации: любой пользователь может отменить свой плановый отпуск, у которого дата начала и окончания — до 31 декабря включительно
- Дни возвращаются в баланс сотрудника

> ⚠️ К уточнению: нужно ли согласование при такой отмене?

### 3.4 Техническая поддержка

Доступ ко всем функциям системы. Детальное описание роли — в следующей итерации ТЗ.

---

### 3.5 Отпуска: просмотр статистики

Отдельная роль. Основной функционал — вкладка с отчётностью по кампаниям:

- Статистика по отменам плановых отпусков:
  - Количество заявок на отмену по статусам (на согласовании / согласована / отклонена)
  - Количество сотрудников, подавших заявки
  - Суммарное количество отменённых дней
- Выгрузка отчёта

## 4. UI / UX-требования

### 4.1 Навигация

- Разделы и функционал разбиты по табам

### 4.2 Отображение отрезков отпуска

- Отрезки планового отпуска отображаются одновременно в двух видах:
  - **Таблица** — структурированный список
  - **Календарь** — визуальное отображение

### 4.3 Флоу планирования

- Поддерживает два пути:
  - Распланировать все дни сразу и отправить
  - Начать, сохранить черновик, вернуться позже
- Кнопка «Отправить на согласование» доступна только когда все дни распределены
- Пользователь должен понимать, сколько дней ещё не распределено

### 4.4 Разграничение планового и внепланового

Ключевая UX-задача: пользователи путают плановый и внеплановый отпуск. Сервис должен активно вести пользователя по правильному пути — через подсказки, контекстные объяснения и чёткое разделение сценариев в интерфейсе.

### 4.5 Статус кампании

Пользователь всегда видит, идёт ли кампания сейчас. Интерфейс адаптируется под состояние кампании: активна / не активна. Пустые стейты и подсказки учитывают текущий контекст.

### 4.6 Эмоциональные элементы *(одобрено)*

- **Countdown** до начала ближайшего согласованного отпуска — размещается на видном месте в интерфейсе сотрудника
- **Виджет-обложка:** пользователь загружает персональное изображение; по умолчанию — заглушка-иллюстрация
